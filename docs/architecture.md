# Architecture Rules

 Business Logic → features/
 Routes → app/
 UI Components → components/
 Backend Clients → server/
 Validation → schemas/
 Types → types/
 Utilities → lib/
 Configuration → config/
 Documentation → docs/
This structure keeps the project scalable, maintainable, interview-ready, and production-ready.
